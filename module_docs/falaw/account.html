falaw.account

Account health probe.

fal_client swallows the JSON body of its 403s, so a locked-account error looks identical to “wrong API key” or “model is busy.” health_check() makes a tiny, idempotent request to the fal storage endpoint and returns a typed AccountStatus so the caller can distinguish:

  • account is locked / unverified → AccountStatus.locked is True

  • bad credentials → AccountStatus.unauthorized is True

  • everything looks fine → AccountStatus.ok is True

  • network / unknown error → all flags False, error populated

Use this before a long render to fail fast with a useful message:

from falaw.account import health_check
status = health_check()
if not status.ok:
    raise SystemExit(status.message_for_user())

falaw.account.health_check(*, probe_url: str = 'https://rest.fal.ai/storage/auth/token?storage_type=fal-cdn-v3', timeout_s: float = 10.0, api_key: str | None = None) → AccountStatus[source]

Probe the fal storage endpoint and return a typed AccountStatus.

Parameters:

  • probe_url – Endpoint to probe. Default is the storage-auth-token endpoint fal_client uses internally; small, idempotent, requires auth.

  • timeout_s – Network timeout.

  • api_key – Optional override of FAL_KEY. Falls back to the env var.

Returns:

AccountStatus with ok=True when the probe succeeded.
