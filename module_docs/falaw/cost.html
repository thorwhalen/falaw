falaw.cost

Cost estimation: ModelRecord.cost_estimate + Scene rollups.

Two surfaces here:

  • estimate_call_cost(record, *, count=1, seconds=None, megapixels=None, tokens=None)() — turn one model’s CostEstimate into a USD figure.

  • estimate_scene_cost(scene, *, ...)() — walk every Shot + Beat in a falaw.Scene and sum the per-call costs that falaw.render_scene() would incur. Returns a structured rollup (per-line + totals) so UIs can show “this render would cost ~$0.42”.

A scene-level estimate is advisory. It’s based on the same pick_model(category, quality_tier=...) lookup that the renderer uses, multiplied by per-line duration / pixel budgets. Real costs vary with model fluctuations and provider billing rules, but this is good enough to gate on (e.g. --budget=1.00) and to surface in muvid status.

falaw.cost.estimate_call_cost(record: ModelRecord, *, count: int = 1, seconds: float | None = None, megapixels: float | None = None, tokens: int | None = None) → float | None[source]

Cost of one fal call against record.

Returns None if the record has no cost_estimate (so callers can distinguish “free” from “unknown”). The unit picked is whatever matches the record’s cost_estimate.kind; mismatched units fall back to the closest-fit default.

falaw.cost.estimate_scene_cost(scene: Scene, *, tts_quality: str = 'balanced', lipsync_quality: str = 'high', shot_quality: str = 'balanced', shots_as_video: bool = False) → CostRollup[source]

Estimate the USD cost of a full render_scene() invocation.

Walks every shot + beat with the same pick_model semantics the renderer uses, then sums per-call costs. Returns a structured CostRollup with per-line breakdowns, plus a list of “skipped” entries the caller should surface (typically: a model with no cost_estimate populated).
